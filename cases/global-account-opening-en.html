Case Study 01

Global Institutional Onboarding and CRM Review System

This project was a bilateral workflow redesign. The issue was not only on the submission side or only on the review side. The system treated both sides as separate problems.

Snapshot

  • Scope: Submission and reviewer workspace
  • Markets: HK, SG, AU, US
  • Outcome: 62% to 89% approval rate

Context

The project supported institutional onboarding across multiple regulated markets. Documentation requirements, field logic, and compliance expectations differed by region, which made the workflow highly complex.

In the previous setup, the submission experience and the reviewer experience were optimized separately, which pushed many errors into later stages and created high rework cost.

Problem definition

  • Errors surfaced too late, reducing first-pass approval.
  • Reviewers had to reconstruct context manually across fragmented information views.
  • The workflow transferred cost across roles instead of reducing it systemically.

Actions

  • Mapped the end-to-end service blueprint, including submission, validation, review, rejection, and resubmission.
  • Redesigned the submission flow with pre-validation, layered error handling, and guided completion.
  • Rebuilt the reviewer workspace around priority, exception detection, and information aggregation.
  • Aligned the team around a single system goal instead of two disconnected screen-level goals.

Service blueprint explicitly shown in Figma

  • User actions: understand requirements, fill information, submit, amend, receive results.
  • Frontend touchpoints: onboarding intro, forms and uploads, error feedback, amendment list, result page.
  • Backend actions: rule diversion, field validation, submission checks, review workflow, result transfer.
  • Support systems: rules library, file system, validation engine, CRM review desk, notification system.

Old vs. new workflow

  • Old workflow issues: scattered rules, lengthy forms, delayed error detection, poor communication, weak result explanation.
  • New workflow strategy: streamlined preparation, stepwise filling, real-time validation, amendment guidance, collaborative review.
  • Submission-side goal: move error detection forward.
  • Review-side goal: improve decision speed through task layering and information aggregation.

Review workspace optimization

  • Submission endpoint: clearer result and field mapping.
  • Review endpoint: layered queues and better task organization.
  • Information aggregation: less manual comparison cost for reviewers.

Reasoning

  • If critical errors only surface in review, the submission experience is not carrying enough decision support.
  • If reviewers need to assemble context manually, the system is under-serving the collaboration layer.
  • If rework happens mostly after handoff, screen-level optimization is insufficient.

Outcome

  • First-pass approval improved from 62% to 89%.
  • Rejection rate reduced by 40%.
  • Review efficiency improved by 35%.

The broader value was a shared team model of treating submission and review as one connected workflow.
